웹에서 빈번하게 발생하는 취약점 세 가지인, SQL Injection 공격, XSS 공격, Command Injection 공격을 대표로 하여 분석을 진행했습니다.

SQL Injection 페이로드를 주입한 후, 서버의 SQL 쿼리를 통해 실행되고, 서버 응답 텍스트를 확인하여 오류 메시지가 포함된 경우 SQL Injection 취약점 발견으로 판단하도록 했습니다

XSS (Cross-Site-Scripting) 페이로드를 주입한 후에는, 주입된 페이로드가 서버 응답 내에 포함되어 있는지를 기준으로 하여 취약점의 유무를 판단하도록 했습니다.

Command Injection의 경우, 주입된 명령어가 실행되었는지 확인하기 위해 시스템 정보가 응답으로 반환되는 경우 Command Injection 취약점이 있다고 판단하도록 설계하였습니다.

각 페이로드를 주입한 후 서버로부터 반환된 응답과 상태코드를 기준으로 취약점이 발견되었는지 여부를 판단하여 각각의 결과를 저장 후 fuzzer_report.pdf에서 확인 가능하도록 했습니다.
